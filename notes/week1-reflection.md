# Week 1 Reflection

# What is Git
 - git is a very useful and very powerful tool used to help developers and teams of developers manage their codebase. 
 - Git is a way to store code on a remote repository that can be accessed by a lot of people, cloned on to each machine for testing and feature development, and is an efficient way to test code prior to adding it back to the main branch. This helps avoid bugs and allows multiple people to be working on something simultaneously.

 # What are Commits? 
 - Commits are the version of code you are sending up to the main branch. 
 - A commit should have a good message that is detailed enough about what it is you are committing. 
 - Commits are a way to stage your code to be merged with the main branch. 

 # What are git pull requests? 
 - a git pull request is a way to propose combining two branches together and allows for team discussion and decision prior to merging. 
 - This ensures good quality and team work.
